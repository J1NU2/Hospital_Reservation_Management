프로젝트 소개(시나리오)
　기간 : 24.10.14 ~ 24.10.24
　소개
　　- 원격으로 병원의 진료 예약을 할 수 있도록 하는 병원 예약 관리 프로그램
　　- Oracle DBMS를 이용한 데이터 관리
　　- Java awt 및 swing을 이용한 GUI 구성
　기능
　　1. 회원가입 (일반회원, 의사)
　　　- 일반 회원은 회원가입을 통해 회원 정보를 등록해야한다.
　　　- 회원가입 시 아이디, 비밀번호, 이름, 주민등록번호, 나이, 성별, 휴대전화번호, 가입날짜의 정보를 저장한다.
　　　- 회원가입 시 비밀번호는 비밀번호 확인을 통해 일치하는지 검사를 진행한다.
　　　- 아이디 중복 생성을 방지하기 위해 아이디와 주민등록번호를 기준으로 중복검사를 진행한다.
　　　- 주민등록번호 입력 시 나이와 성별이 자동으로 입력된다.
　　　　: 나이는 만나이를 기준으로 입력된다.
　　　　: 성별은 주민등록번호 뒷자리 1번째 자리를 기준으로 입력된다.
　　　- 가입날짜는 입력받는 것이 아닌 회원가입 성공 시 현재일을 기준으로 자동 등록된다.
　　　- 의사의 정보 입력 시 미리 정의된 인증번호를 입력하고 정보를 입력해야한다.
　　　- 의사 정보는 이름, 의사번호, 비밀번호, 전공의 정보를 저장한다.
　　　- 의사 정보 중복 생성을 방지하기 위해 의사번호를 기준으로 중복검사를 진행한다.
　　　- 의사 정보 등록 시 비밀번호는 비밀번호 확인을 통해 중복검사를 진행한다.
　　　- 전공은 선택받도록 한다.
　　　　: 내과, 일반외과, 정형외과, 소아과, 안과, 이비인후과, 피부과, 비뇨기과, 정신과, 성형외과 중 택 1
　　　- 만약, 하나의 정보라도 미입력된다면 회원가입이 진행되지 않도록 한다.
　　2. 로그인 (일반회원, 의사)
　　　- 일반회원 로그인 시 회원가입된 아이디, 비밀번호를 입력받아 로그인한다.
　　　- 의사 로그인 시 미리 정의된 인증번호를 입력한 뒤 저장된 의사번호, 비밀번호를 입력받아 로그인한다.
　　3. 예약하기
　　　- 예약 시 현재 로그인한 사용자의 정보(주민등록번호)를 가져온다.
　　　- 예약 시 선택한 예약날짜, 예약시간의 정보를 저장한다.
　　　- 예약 시 선택한 의사의 정보(의사번호)를 저장한다.
　　　　: 사용자(주민등록번호), 의사(의사번호), 예약날짜, 예약시간
　　　- 진료가 완료되기 전까지 중복해서 예약을 할 수 없도록 한다.
　　　- 한명의 의사에게 예약할 때 날짜와 시간이 중복되지 않도록 이미 예약한 날짜와 시간이 있다면 예약을 할 수 없도록 한다.
　　4. 예약 목록 확인/취소하기
　　　- 예약 목록은 번호, 예약자의 정보(주민등록번호), 담당의사의 정보(의사번호), 예약날짜, 예약시간, 진료여부, 증상메모, 취소사유의 정보를 가진다.
　　　- 일반회원 기준
　　　　: 제일 상단에 자신(회원)의 이름, 주민등록번호가 표시된다.
　　　　: 만약 자신이 현재 예약한 내역이 있다면 해당 예약 내역을 보여준다.
　　　　: 현재 예약 내역은 의사이름, 의사번호, 예약날짜, 예약시간의 정보가 표시된다.
　　　　: 현재 예약한 내역은 취소할 수 있다.
　　　　: 현재까지 예약한 모든 내역을 확인할 수 있다.
　　　　: 현재까지 예약한 모든 내역에는 번호, 의사이름, 의사번호, 예약날짜, 예약시간, 증상메모, 취소사유의 정보가 표시된다.
　　　- 의사 기준
　　　　: 제일 상단에 자신(의사)의 이름, 의사번호가 표시된다.
　　　　: 자신에게 예약한 모든 내역을 확인할 수 있다.
　　　　: 자신에게 예약한 모든 내역에는 번호, 예약한 사람의 이름, 주민등록번호, 나이, 성별, 예약날짜, 예약시간, 진료여부, 증상메모의 정보가 표시된다.
　　　　: 내역 중 하나를 선택하여 증상메모를 입력하여 저장할 수 있다.
　　　　: 입력된 증상 메모는 수정이 가능하다.
　　　　: 내역 중 하나를 선택하여 예약취소를 할 수 있다.
　　　　: 예약 취소 시 취소사유를 입력해야 한다.
